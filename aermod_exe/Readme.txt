README – Інструкція запуску моделі AERMOD

---------------------------------------------
1. Необхідні дані
---------------------------------------------
Для роботи AERMOD потрібні такі вхідні дані:
- Метеорологічні метадані (згенеровані AERMET)
- Дані рельєфу та рецепторів (згенеровані AERMAP)
- Дані про будівлі (опціонально)
- Налаштування директорії AERMOD

---------------------------------------------
2. Підготовка метеоданих – AERMET
---------------------------------------------
Файли AERMET розташовані у каталозі AERMET/.  
Процес складається з двох етапів:  
1) Зчитування сирих даних  
2) Об’єднання та формування вихідних файлів

Етап 1 – Зчитування даних (aermet1.inp):
- upperair – висотні sounding дані (наприклад, 14735-92.fsl)  
- surface – дані метеостанції NWS у форматі CD144 (14737.dat)  
- onsite – локальні вимірювання (mcospfl.dat)  
- вказати формати полів, діапазони, обробку пропусків, контроль якості (qaout, range, no_missing тощо)

Етап 2 – Об’єднання (aermet2.inp):
- вибір QA-файлів (alb92-93.oqa, 14737.oqa, mcospfl.oqa)  
- часовий діапазон (xdates)  
- координати місця (location)  
- висота вимірювання вітру (nws_hgt wind 6.1)  
- параметри місцевості: альбедо, roughness, Bowen ratio

Результат роботи AERMET:
- aermet.pfl  
- aermet.sfc

Команди для запуску:
cd AERMET
aermet aermet1.inp
aermet aermet2.inp

---------------------------------------------
3. Підготовка рельєфу та рецепторів – AERMAP
---------------------------------------------
Файли AERMAP розташовані у каталозі AERMAP/.  
На вхід подається файл aermap.inp, що містить:

- TITLEONE – назва проєкту  
- DATATYPE – тип вхідних даних рельєфу (наприклад, DEM1)  
- DATAFILE – DEM-файли (NEWARK-W.DEM, NEWARK-E.DEM)  
- DOMAINXY, ANCHORXY – межі області та сітка  
- RUNORNOT – команда запуску (RUN)  

Джерела (LOCATION STACK1–STACK6):
- тип – POINT  
- координати X, Y, Z  
- висота місцевості визначається автоматично з DEM  

Рецептори (DISCCART):
- координати X, Y, Z (або TERRHGTS PROVIDED)  
- можливе додавання додаткових точок: MET TOWER, SODAR-сайт

Результат роботи AERMAP:
- aermap.rec  
- aermap.src

Команда для запуску:
cd AERMAP
aermap

---------------------------------------------
4. Запуск основної моделі – AERMOD
---------------------------------------------
Підготувати файл aermod.inп, який містить основні блоки:

CO STARTING / FINISHED – службова інформація:
- TITLEONE, TITLETWO – назви моделювання  
- MODELOPT – режими (наприклад, default conc)  
- AVERTIME – періоди усереднення (1, 3, 24 год, річний)  
- POLLUTID – забруднювач (SO2)  
- RUNORNOT – run  
- ERRORFIL – журнал помилок  

SO STARTING / FINISHED – джерела викидів:
- LOCATION – координати X, Y, Z  
- SRCPARAM – параметри (QS, HS, TS, VS, DS)  
- HOUREMIS – файл годинних емісій  
- BUILDHGT, BUILDWID, BUILDLEN – параметри будівель  
- XBADJ, YBADJ – зміщення будівель  
- SRCGROUP – групування джерел  

RE STARTING / FINISHED – підключення рецепторів (файл aermap.rec)  

ME STARTING / FINISHED – підключення метеоданих (aermet.sfc, aermet.pfl, станційні коди, рік, PROFBASE)

OU STARTING / FINISHED – результати:
- RECTABLE, MAXTABLE – таблиці результатів  
- SUMMFILE – зведений файл (aermod.sum)

Команда для запуску:
cd AERMOD
aermod

---------------------------------------------
5. Підсумок
---------------------------------------------
Порядок виконання:
1. Підготувати метеодані (AERMET)  
2. Підготувати рельєф та рецептори (AERMAP)  
3. Запустити основний розрахунок (AERMOD)

Вихідні результати – у файлі aermod.sum  
Журнали та службові файли – у відповідних підкаталогах.
